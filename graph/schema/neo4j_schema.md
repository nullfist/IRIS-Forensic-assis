# IRIS Neo4j Graph Schema

## Overview

This document defines the complete graph schema used by IRIS for storing entity relationships, attack paths, and investigation context.

## Node Labels

### Host
Represents computer systems, servers, and network devices in the environment.

```cypher
CREATE CONSTRAINT host_id_unique IF NOT EXISTS FOR (h:Host) REQUIRE h.host_id IS UNIQUE;

Node Properties:
- host_id: string (unique identifier, e.g., "host:workstation-01")
- name: string (hostname)
- os: string (operating system)
- domain: string (Active Directory domain)
- ip_addresses: list[string] (all known IP addresses)
- risk_score: float (0.0 - 1.0)
- first_seen: datetime
- last_seen: datetime
- investigation_id: string (for multi-tenant isolation)
```

### User
Represents user accounts (domain and local).

```cypher
CREATE CONSTRAINT user_id_unique IF NOT EXISTS FOR (u:User) REQUIRE u.user_id IS UNIQUE;

Node Properties:
- user_id: string (unique identifier, e.g., "user:corp\\jsmith")
- name: string (username)
- domain: string (domain name)
- email: string (email address)
- risk_score: float (0.0 - 1.0)
- first_seen: datetime
- last_seen: datetime
- investigation_id: string
```

### Process
Represents running processes on endpoints.

```cypher
CREATE CONSTRAINT process_id_unique IF NOT EXISTS FOR (p:Process) REQUIRE p.process_id IS UNIQUE;

Node Properties:
- process_id: string (unique identifier, typically ProcessGuid)
- image: string (full path to executable)
- image_name: string (executable name only)
- pid: integer (process ID)
- command_line: string (full command line)
- host: string (hostname where process runs)
- user: string (user context)
- integrity_level: string (Low, Medium, High, System)
- hashes: map (file hashes: SHA256, MD5, etc.)
- risk_score: float (0.0 - 1.0)
- start_time: datetime
- end_time: datetime
- investigation_id: string
```

### File
Represents files on the filesystem.

```cypher
CREATE CONSTRAINT file_id_unique IF NOT EXISTS FOR (f:File) REQUIRE f.file_id IS UNIQUE;

Node Properties:
- file_id: string (unique identifier)
- path: string (full file path)
- name: string (filename)
- extension: string (file extension)
- size_bytes: integer
- hashes: map (file hashes)
- host: string (hostname)
- risk_score: float (0.0 - 1.0)
- first_seen: datetime
- investigation_id: string
```

### IP
Represents IP addresses (internal and external).

```cypher
CREATE CONSTRAINT ip_id_unique IF NOT EXISTS FOR (ip:IP) REQUIRE ip.ip_id IS UNIQUE;

Node Properties:
- ip_id: string (unique identifier, e.g., "ip:192.168.1.105")
- address: string (IP address)
- type: string (internal, external, loopback)
- geo_location: map (country, city, lat, lon for external IPs)
- risk_score: float (0.0 - 1.0)
- first_seen: datetime
- last_seen: datetime
- investigation_id: string
```

### Domain
Represents DNS domain names.

```cypher
CREATE CONSTRAINT domain_id_unique IF NOT EXISTS FOR (d:Domain) REQUIRE d.domain_id IS UNIQUE;

Node Properties:
- domain_id: string (unique identifier, e.g., "domain:evil.com")
- name: string (domain name)
- category: string (legitimate, suspicious, known-malicious)
- risk_score: float (0.0 - 1.0)
- first_seen: datetime
- last_seen: datetime
- investigation_id: string
```

### Alert
Represents detection alerts generated by the system.

```cypher
CREATE CONSTRAINT alert_id_unique IF NOT EXISTS FOR (a:Alert) REQUIRE a.alert_id IS UNIQUE;

Node Properties:
- alert_id: string (unique identifier)
- rule_id: string (detection rule identifier)
- family: string (alert family/category)
- title: string (alert title)
- severity: string (info, low, medium, high, critical)
- status: string (new, in_review, confirmed, suppressed, closed)
- phase: string (MITRE ATT&CK phase)
- risk_score: float (0.0 - 1.0)
- confidence: float (0.0 - 1.0)
- created_at: datetime
- updated_at: datetime
- investigation_id: string
```

### Event
Represents normalized security events.

```cypher
CREATE INDEX event_timestamp IF NOT EXISTS FOR (e:Event) ON (e.timestamp);

Node Properties:
- event_id: string (unique identifier)
- investigation_id: string
- source: string (sysmon, evtx, pcap)
- category: string (process, network, file, registry)
- event_type: string (specific event type)
- timestamp: datetime
- host: string
- user: string
- severity: string
- attack_phase: string
```

## Relationship Types

### Process Relationships

```cypher
(:Process)-[:RUNS_ON {timestamp}]->(:Host)
(:Process)-[:RUN_AS {timestamp}]->(:User)
(:Process)-[:SPAWNED {timestamp, inheritance_level}]->(:Process)
(:Process)-[:CONNECTED_TO {port, protocol, bytes_sent, bytes_received, timestamp}]->(:IP)
(:Process)-[:CONNECTED_TO {port, protocol, timestamp}]->(:Domain)
(:Process)-[:ACCESSED {operation, timestamp}]->(:File)
(:Process)-[:MODIFIED {operation, timestamp}]->(:File)
(:Process)-[:CREATED {timestamp}]->(:File)
(:Process)-[:MODIFIED_REGISTRY {operation, key_path, timestamp}]->(:RegistryKey)
```

### Host Relationships

```cypher
(:Host)-[:COMMUNICATED_WITH {protocol, port, bytes, timestamp}]->(:Host)
(:Host)-[:HOSTS {timestamp}]->(:Process)
(:Host)-[:CONTAINS {path, timestamp}]->(:File)
```

### User Relationships

```cypher
(:User)-[:LOGGED_INTO {logon_type, timestamp, source_ip}]->(:Host)
(:User)-[:USED {timestamp}]->(:Process)
(:User)-[:OWNS {timestamp}]->(:File)
```

### Alert Relationships

```cypher
(:Alert)-[:RELATED_TO {confidence}]->(:Process)
(:Alert)-[:RELATED_TO {confidence}]->(:Event)
(:Alert)-[:AFFECTS {severity}]->(:Host)
(:Alert)-[:INVOLVES {role}]->(:User)
(:Alert)-[:TARGETS {confidence}]->(:IP)
(:Alert)-[:TARGETS {confidence}]->(:Domain)
```

### Attack Path Relationships

```cypher
(:Alert)-[:PART_OF {sequence_order}]->(:Alert)  // Alert chain
(:Process)-[:PART_OF_ATTACK {phase, sequence_order}]->(:Process)  // Attack chain
```

## Indexes and Constraints

```cypher
// Unique constraints
CREATE CONSTRAINT host_id_unique IF NOT EXISTS FOR (h:Host) REQUIRE h.host_id IS UNIQUE;
CREATE CONSTRAINT user_id_unique IF NOT EXISTS FOR (u:User) REQUIRE u.user_id IS UNIQUE;
CREATE CONSTRAINT process_id_unique IF NOT EXISTS FOR (p:Process) REQUIRE p.process_id IS UNIQUE;
CREATE CONSTRAINT file_id_unique IF NOT EXISTS FOR (f:File) REQUIRE f.file_id IS UNIQUE;
CREATE CONSTRAINT ip_id_unique IF NOT EXISTS FOR (ip:IP) REQUIRE ip.ip_id IS UNIQUE;
CREATE CONSTRAINT domain_id_unique IF NOT EXISTS FOR (d:Domain) REQUIRE d.domain_id IS UNIQUE;
CREATE CONSTRAINT alert_id_unique IF NOT EXISTS FOR (a:Alert) REQUIRE a.alert_id IS UNIQUE;

// Performance indexes
CREATE INDEX event_timestamp_idx IF NOT EXISTS FOR (e:Event) ON (e.timestamp);
CREATE INDEX process_image_idx IF NOT EXISTS FOR (p:Process) ON (p.image);
CREATE INDEX host_name_idx IF NOT EXISTS FOR (h:Host) ON (h.name);
CREATE INDEX user_name_idx IF NOT EXISTS FOR (u:User) ON (u.name);
CREATE INDEX alert_severity_idx IF NOT EXISTS FOR (a:Alert) ON (a.severity);
CREATE INDEX alert_status_idx IF NOT EXISTS FOR (a:Alert) ON (a.status);
CREATE INDEX ip_address_idx IF NOT EXISTS FOR (ip:IP) ON (ip.address);
CREATE INDEX investigation_id_idx IF NOT EXISTS FOR (n) ON (n.investigation_id);
```

## Common Traversal Queries

### Find All Processes Spawned by a Suspicious Process
```cypher
MATCH (p:Process {process_id: $process_id})-[:SPAWNED*]->(child:Process)
RETURN child
ORDER BY child.start_time
```

### Find Attack Paths Between Two Hosts
```cypher
MATCH path = allShortestPaths(
  (h1:Host {name: $source_host})-[*..5]-(h2:Host {name: $target_host})
)
WHERE ALL(n IN nodes(path) WHERE n.investigation_id = $investigation_id)
RETURN path
```

### Find All Alerts Affecting a User
```cypher
MATCH (a:Alert)-[:INVOLVES]->(u:User {user_id: $user_id})
WHERE a.investigation_id = $investigation_id
RETURN a
ORDER BY a.created_at DESC
```

### Find Lateral Movement Patterns
```cypher
MATCH path = (h1:Host)-[:COMMUNICATED_WITH*2..5]->(h2:Host)
WHERE h1.name <> h2.name
  AND ALL(rel IN relationships(path) WHERE rel.port = 445 OR rel.port = 135)
RETURN path
```

### Find Credential Dumping Indicators
```cypher
MATCH (p:Process)-[:ACCESSED]->(f:File)
WHERE f.path CONTAINS 'lsass' OR f.path CONTAINS 'ntds.dit'
  OR p.command_line CONTAINS 'sekurlsa'
  OR p.command_line CONTAINS 'mimikatz'
RETURN p, f
```

### Build Full Attack Narrative
```cypher
MATCH (start:Alert {alert_id: $initial_alert_id})
MATCH path = (start)-[*..10]-(n)
WHERE ALL(rel IN relationships(path) WHERE rel.investigation_id = $investigation_id)
RETURN path
ORDER BY length(path) DESC
LIMIT 10
```

## Data Retention and Cleanup

```cypher
// Archive old investigations
MATCH (n)
WHERE n.investigation_id = $old_investigation_id
  AND n.last_seen < datetime() - duration('P90D')
DETACH DELETE n

// Clean up temporary data
MATCH (e:Event)
WHERE e.timestamp < datetime() - duration('P30D')
DETACH DELETE e
